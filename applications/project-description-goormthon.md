## 답변 (줄글, 약 500자)

구름톤 17기 in JEJU에서 진행한 '삼춘이랑'은 제주의 독거노인과 청년·여행객을 잇는 커뮤니티 매칭 서비스입니다. 초고령 사회에 진입한 제주에서 어르신의 남는 방과 진짜 제주를 경험하고 싶은 여행객을 연결해, 어르신에게는 따뜻한 동행과 부가 수입을, 여행객에게는 현지 경험을 주자는 아이디어에서 출발했습니다. 저는 Next.js(App Router) 기반 웹 프론트엔드를 맡아 기획 단계부터 참여했고, 짧은 해커톤 기간 안에 서비스가 실제로 돌아가도록 페이지 구조와 사용자 흐름을 설계·구현했습니다. 서버 상태는 TanStack Query로 캐싱하고 즐겨찾기·사용자 정보 같은 전역 상태는 Zustand로 분리해 관리했으며, 디자이너와 실시간으로 소통하며 Figma 시안을 Vapor UI 기반 반응형 컴포넌트로 옮겼습니다. 특히 백엔드 개발이 늦어지는 상황에서, 백엔드 담당자와 미리 API 명세를 맞춘 뒤 이를 스위치로 켜고 끌 수 있는 가짜 데이터 계층으로 만들어, 프론트엔드가 백엔드에 종속되지 않고 먼저 배포·검증되도록 했습니다. 이 구조 덕분에 팀 전체가 병렬로 작업하며 개발 속도를 높일 수 있었습니다. 기획·디자인·개발이 동시에 굴러가는 해커톤 환경에서 다른 파트와 활발히 조율하며, 프론트엔드를 끝까지 책임지고 완성한 프로젝트였습니다.

## 참고 (근거가 된 실제 기여 내역)

- 팀 마음충전소 (구름톤 17기 in JEJU) / 역할: Frontend
- Live: https://samchoon-irang.vercel.app
- Next.js App Router 페이지 구조 설계 및 라우팅
- TanStack Query(서버 상태) + Zustand(전역 상태) 분리 구성
- 디자이너 Figma 컴포넌트 + Vapor UI 연동 → 반응형 디자인 시스템 구축
- Open API Generator로 Swagger 기반 API 호출 로직 자동 생성
- SSG로 숙소 상세 페이지 프리렌더링 + Metadata API 링크 공유 미리보기
- 환경변수 스위치 기반 Mock API Layer 구성 → 백엔드 종속성 없이 단독 배포
- LocalStorage로 스플래시 반복 노출 방지 등 UX 디테일 개선
- 지도 필터 인터랙션, toast, 메모리 누수 개선 등
