## Q1. 사용 가능한 프레임워크/라이브러리(React, Vue 등)를 나열하고, 그중 가장 자신 있는 기술을 하나 골라 이유를 설명해주세요.(공란 포함 최대 300자)

React, Next.js, TypeScript, TanStack Query, Framer Motion, Tailwind CSS, Firebase, Flutter
가장 자신 있는 기술은 React입니다. 4개월간 React와 TypeScript로 7개 서비스를 기획부터 배포까지 완성했습니다. Kickoff에서는 TanStack Query로 800개 리그 데이터의 캐싱 전략을 설계하고, React.memo와 useMemo로 렌더링을 최적화했습니다. 반복적인 경험을 통해 컴포넌트 설계부터 상태 관리, 성능 최적화까지 체득한 기술입니다.

## Q2. 최근 6개월 내에 가장 많은 시간을 드려 해결했던 기술적 문제를 작성해주세요.(공란 포함 최대 300자)

> 어떤 상황과 원인에서 발생했는지, 해결 방법과 그 방법을 선택한 이유, 그리고 이 경험을 통해 배우거나 성장한 점을 구체적으로 작성해주세요.

두바이 왕자에서 Firebase 실시간 동시 접속자 UI 구현에 가장 많은 시간을 들였습니다. React에서 onValue로 접속자 수를 구독하되, 접속 +1 퇴장 -1의 단순 카운터로 시작했으나, 브라우저를 닫으면 -1이 실행되지 않아 수치가 누적되었고, onDisconnect를 적용해도 네트워크 불안정 시 지연되었습니다. 최종적으로 유저별 세션 키 기반 Presence 구조로 전환하고, 커스텀 훅으로 로직을 캡슐화해 재사용 가능하게 설계했습니다. 세 번의 구조 변경을 거치며 엣지 케이스를 먼저 고려하는 습관을 갖게 되었습니다.

## Q3. 현재 관심 있는 프론트엔드 기술이나 개념을 작성해주세요.(공란 포함 최대 300자)

> 관심을 갖게 된 이유와 실제 프로젝트에 적용해본 경험이 있다면 그로 인해 개선된 부분을 함께 작성해주세요.

웹 성능 최적화와 사용자 경험 개선에 관심이 있습니다. 아무리 좋은 기능도 느리면 사용자가 떠나기 때문입니다. 포트폴리오에서 이미지를 WebP로 변환해 90% 용량을 줄이고, Next.js의 코드 스플리팅과 초기 로딩을 최적화했습니다. 두바이 왕자에서는 실시간 데이터와 파티클 애니메이션이 동시에 동작하는 환경에서 GPU 가속을 활용해 부드러운 인터랙션을 구현했습니다. 최근에는 Core Web Vitals, React Server Components, Streaming SSR 등 렌더링 성능을 개선하는 기술들을 학습하고 있습니다.

## Q4. 선호하거나 관심 있는 디자인 시스템을 작성해주세요.(공란 포함 최대 300자)

> 선택한 구체적인 이유와 해당 디자인 시스템의 개선 또는 보완이 필요하다고 생각하는 부분을 함께 작성해주세요.

Framer Motion 기반의 모션 디자인 시스템에 관심이 있습니다. 디자이너가 설계한 전환 효과를 코드로 정확히 구현하려면 선언적으로 애니메이션을 제어할 수 있는 도구가 필수적이기 때문입니다. 포트폴리오에서 whileInView, AnimatePresence로 스크롤 진입과 모달 전환을 구현하며 디자인 의도를 코드로 옮기는 경험을 했습니다. 보완이 필요한 부분은 모션 규칙의 체계화입니다. 진입·퇴장·호버 등 상황별로 정의해 모션 토큰처럼 관리하면 일관된 인터랙션을 유지할 수 있다고 생각합니다.

## Q5. 구름톤 참여 동기를 작성해주세요.(공란 포함 최대 300자)

지난 4개월간 7개 서비스를 혼자 기획부터 배포까지 완성했지만, 대부분 1인 개발이었기에 다양한 직군과의 협업 경험이 부족하다는 한계를 느꼈습니다. 저는 지난 해커톤에서 2인 팀으로 장려상을 수상하며 협업의 가치를 체감했는데, 단 두 명이서도 서로의 관점이 합쳐지니 혼자서는 생각하지 못한 방향으로 서비스가 발전했습니다. 디자이너, 기획자, 개발자까지 함께한다면 그 시너지는 비교할 수 없을 것이라 확신합니다. 저는 해커톤 기간 동안 정해진 시간 안에 빠르게, 퀄리티 높은 프로덕트를 구현하는 실행력으로 팀에 기여하고 싶어 지원합니다.

## Q6. 구름톤을 통해 어떤 부분의 '성장'을 기대하고 있는지, 있다면 작성해주세요.(공란 포함 최대 300자)

두 가지 성장을 기대합니다. 첫째, 회사에서 얻기 어려운 협업 경험입니다. 저는 건국대학교를 휴학 후 현재는 산업기능요원으로 재직중에 있습니다. 그렇다 보니 실무에서는 정해진 요구사항을 구현하는 데 집중하고 있습니다. 하지만 해커톤은 아이디어부터 함께 고민하고 힘을 합쳐 하나의 서비스를 완성합니다. 기술 스택은 늘었지만 다른 사람의 관점에서 배우는 경험은 부족했기에, 이 부분을 채우고 싶습니다. 둘째, 제한된 시간 안에서의 판단력입니다. 해커톤은 핵심을 빠르게 파악, 실행하는 능력이 중요합니다. 이러한 부분을 채우고 싶습니다.

## Q7. 협업 시 지키는 본인만의 규칙이나 전략은 무엇인가요? 갈등 상황이 발생했을 때의 해결 방법도 함께 알려주세요.(공란 포함 최대 500자)

저의 협업 규칙은 "솔직하게, 하지만 존중하면서 대화한다"입니다.
팀 프로젝트에서 갈등이 생기는 경우를 보면, 대부분 의견 차이가 아니라 감정싸움으로 번지면서 문제가 커집니다. 같은 말이라도 표현 방식에 따라 결과가 완전히 달라진다고 생각합니다. 그래서 저는 모르는 것은 모른다고 솔직히 말하고, 의견이 다를 때는 감추지 않되 반드시 근거와 함께 이야기합니다.
저는 이를 직접 경험했습니다. 관광 코스를 추천해주는 프로젝트에서 추천 정확도와 사용자 경험 사이에서 팀원과 의견이 갈렸는데, 서로 "왜 그렇게 생각하는지" 근거를 솔직하게 공유하자 한쪽을 포기하는 것이 아니라 다른 API를 도입해 추천 범위를 먼저 축소하는 제3의 방안이 나왔습니다. 양쪽 모두 만족하는 결과를 얻었고 높은 퀄리티의 프로덕트를 만들어 냈습니다.
애매한 부분을 넘어가지 않고 바로 확인하는 것, 상대를 존중하되 핵심은 솔직히 말하는 것이 팀의 신뢰를 쌓는 기본이라고 생각합니다.

## Q8. 진행하신 프로젝트 중 하나를 선택하여 설명해 주시고, 그 경험으로 인해 얻을 수 있었던 결과를 작성해주세요.(공란 포함 최대 500자)

Kickoff 프로젝트를 선택하겠습니다. 800개 이상의 축구 리그 데이터를 실시간으로 보여주는 SPA를 React와 TypeScript로 개발해 토스 앱인토스에 배포했습니다.
가장 큰 도전은 방대한 외부 API 데이터를 효율적으로 관리하는 것이었습니다. 각 컴포넌트에서 개별적으로 fetch를 호출하니 동일 데이터의 중복 요청이 발생하고, 페이지 전환마다 로딩이 반복되었습니다. TanStack Query를 도입해 실시간 스코어는 staleTime 30초, 리그 순위는 5분으로 데이터 특성별 캐싱 전략을 분리했고, queryKey를 리그·날짜·팀 단위로 세분화해 한 화면의 갱신이 다른 화면에 영향을 주지 않도록 설계했습니다. React.memo와 useMemo로 불필요한 리렌더링을 방지하고 스켈레톤 UI로 속도도 개선했습니다.
이 경험을 통해 상태 관리는 단순히 라이브러리 선택이 아니라, 데이터의 성격과 갱신 주기를 분석하고 적절한 전략을 설계하는 것이 핵심이라는 것을 배웠습니다.
